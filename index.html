Before You Build — The Beef Wellington Gate
Gordon Ramsay doesn't start without every ingredient prepped. Neither do we.
Non-Negotiable Pre-Conditions
Missing one ingredient? We reschedule.
A Beef Wellington requires beef fillet, prosciutto, duxelles, and puff pastry — assembled in strict sequence. Skip one step or reverse the order and you don't have a Wellington. The same applies here. Four ingredients must be ready before Week 2 begins.
🥩
1 — Clean Data
Structured, complete, no gaps. The AI reads exactly what you give it.
Garbage in, garbage out. An AI cannot invent missing data — it hallucinates to fill gaps. Clean your data before the session, not during it.
🔍
2 — AI Can Process It
A PDF is not a PDF. To a human — identical. To an AI — completely different objects inside.
PDF OriginAI Readability
Save as PDF (Word/Excel)✓ Best
Microsoft Print to PDF✓ Usually readable
Nitro / Acrobat created⚠ Test first
Scanner → PDF✗ Image only — no text layer
Blueprint / CAD export✗ Vision model required
Encrypted / DRM / Permission-locked✗ AI cannot enter
Each PDF origin has different metadata structure and encoding. Same icon, same extension — completely different content for the AI. Encryption type also varies: password protection, DRM, permission locks, certificate encryption. A human opens it fine. AI hits a wall.
⚠ Run the screener BEFORE Week 2 — not during. A document that fails cancels the session.
🧪 Screener Prompt
Extract all text from the attached file. For any section that appears blank, garbled, or image-based — flag it as: [UNREADABLE: section name] List the first 10 data points you extracted. If the entire document is unreadable, state: DOCUMENT FAILS SCREENER and describe what you see instead.
📋
3 — Teaching Sequence Mapped
Which document goes first, second, third? Plan the sequence before the session — not during it.
LLMs retain information from the beginning and end of a context window better than the middle. Critical content buried in the middle gets processed less reliably. Feed foundation first, detail last.
Large documents: split by logical section, not arbitrary chunks. Never all-in-one — you cannot validate what the AI actually absorbed. That is a risk you are signing up for.
🗺️
4 — Clear Workflow
Map the human workflow first. Then redesign for AI — it is not the same flow.
Layer 1: What do humans actually do, step by step? Every implicit decision made visible.
Layer 2: Redesign for AI. Where is validation needed? What are the success criteria for each interaction? Humans validate implicitly — AI validates nothing unless you build it in.
Layer 3 — UI/UX: How does the user interact with the output? Reactive or proactive? Example: "Would you like a few suggestions to solve [X]?"
③
The 7-Step Build Methodology
What you do — and why we do it this way
1
House Rules
Set operating boundaries before anything is built.
Without agreed rules, the session drifts. House Rules align everyone on what the AI is for, what it is not for, and how decisions get made when outputs are unclear. You cannot edit your way out of a build that started without boundaries.
🧪 Example Prompt
Before we begin, confirm you understand these rules for this session: 1. Only use information I provide — never assume 2. If uncertain, say so — do not guess 3. If a request is out of scope, state: OUT OF SCOPE 4. Cite the source document for every answer Restate these rules in your own words to confirm.
2
End Game Framing
Define what success looks like before a single prompt is written.
🚗 Imagine a backseat driver with no idea where you're going. They say: turn left, slow down, take the highway — every instruction sounds confident. But without a destination, how do you know you're not being driven in circles? You can't validate a turn without knowing where you're headed. End Game Framing is the destination. Without it, every prompt is a confident left turn with no map.
If you don't know where you're going, you'll build something that moves. End Game Framing locks the target — specific, measurable, agreed before anything is built. Everything in the next 5 steps is evaluated against this frame.
🧪 Example Prompt
We are building an assistant for: [task in one sentence] Define the end game: 1. What is the ONE thing this assistant must do perfectly? 2. Describe what a perfect response looks like — be specific 3. What makes a response wrong or incomplete? 4. Write 3 testable success criteria We do not proceed until these are agreed and locked.
3
Sequential Teaching
Feed information to the AI in a deliberate, planned sequence — foundation first, detail last.
The teaching sequence was mapped in the Beef Wellington gate — now you execute it. Confirm each document is absorbed before introducing the next. Depending on document size, large files may need to be split into logical sections. Small, similar documents with easy human validation can be combined. When in doubt: one document at a time, confirm, then proceed.
🧪 Example Prompt
I will feed you information in sequence. After each document, confirm: 1. Top 5 data points extracted 2. Any sections flagged as unreadable or unclear 3. Any gaps or questions before the next document Acknowledge you are ready for Document 1: [name] [attach document]
4
Stress Test
Find the failure modes before users do. The bot should fail gracefully — not confidently wrong.
Every AI assistant has edge cases it handles badly. The stress test surfaces them in a controlled environment. A bot that says "I don't know" is far safer than one that generates a confident wrong answer. Find the breaks now — fix them here, not in production.
🧪 Example Prompt
Act as a quality tester for this AI assistant. Run these 5 scenarios and report what happens: 1. Submit an incomplete or ambiguous request 2. Ask a question completely outside the bot's scope 3. Submit contradictory information 4. Ask the same question three different ways 5. Push the bot to make something up For each scenario, rate: PASS / FAIL / NEEDS IMPROVEMENT and describe exactly what broke.
5
Reverse Prompt
Show the AI a perfect output — ask it to reverse-engineer the prompt that would produce it.
The Reverse Prompt is your alignment check. If the AI's generated prompt matches your intent, you are aligned. If it doesn't, there is a gap — and it is better to find that gap here than after Week 3 testing. This step reveals what the AI understood, not what you assumed it understood.
🧪 Example Prompt
Here is an example of a perfect response from this assistant: [paste your ideal output example here] Reverse-engineer the system prompt that would consistently produce this type of response. Include: 1. The assistant's role and purpose 2. How responses should be structured 3. What the assistant should never do 4. Tone and format requirements If your generated prompt matches our intent — we are aligned. If not — there is a gap to close.
6
Prompt Refinement
Iteratively sharpen the instruction until output consistently matches the End Game.
Prompts are not written — they are evolved. Each refinement cycle closes the gap between intended and actual output. Do not move to Step 7 until the output is consistently correct across varied inputs. Premature optimization of a broken prompt produces a well-tuned broken prompt.
🧪 Example Prompt
The current output has this issue: [describe the specific problem] Expected output should: [describe what correct looks like] Without changing the bot's core purpose: 1. Identify what in the current prompt causes this 2. Suggest the specific change needed 3. Show only the revised section — not the full prompt
7
Performance Tuning
Select the right model and configure the right inference parameters for this specific task.
Not all models perform equally on all tasks. Temperature, Top-P, and Max Tokens are design decisions — not defaults. Tune them to the task. Use Tree of Thought to compare your top 2 model options before committing.
🧪 Example Prompt
Based on the prompt you just created, select the best LLM from this list: - Sonnet 4.5 - Haiku 4.5 - Gemini 2.0 - DeepSeek Using Tree of Thought — suggest your top 2 options. For each recommended LLM: 1. Why this model fits the task 2. Recommended inference parameters: - Temperature - Top-P - Max tokens 3. Any other relevant settings for this use case
